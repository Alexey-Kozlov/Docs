скачиваем kafka:

docker pull bitnami/kafka:latest
docker pull provectuslabs/kafka-ui:v0.7.2

kafka (SSH)

Должен быть  загружен плагин - docker plugin install vieux/sshfs

Монтируем volume:
docker volume create --driver vieux/sshfs -o sshcmd=alex@192.168.1.12:/home/alex/data/Kafka_Local -o password=1 -o allow_other="" -o default_permissions="" -o port=22 Kafka_Local

Запускаем контейнер:
docker run -d --name kafka -p 9092:9092 -p 9093:9093 -e KAFKA_ENABLE_KRAFT=yes -e KAFKA_CFG_PROCESS_ROLES=broker,controller -e KAFKA_CFG_CONTROLLER_LISTENER_NAMES=CONTROLLER -e KAFKA_CFG_LISTENERS=PLAINTEXT://:9092,CONTROLLER://:9093 -e KAFKA_CFG_LISTENER_SECURITY_PROTOCOL_MAP=CONTROLLER:PLAINTEXT,PLAINTEXT:PLAINTEXT -e KAFKA_CFG_ADVERTISED_LISTENERS=PLAINTEXT://host.docker.internal:9092 -e KAFKA_CFG_BROKER_ID=1 -e KAFKA_CFG_CONTROLLER_QUORUM_VOTERS=1@localhost:9093 -e ALLOW_PLAINTEXT_LISTENER=yes -e KAFKA_KRAFT_CLUSTER_ID=kafka_local -e KAFKA_CFG_NODE_ID=1 -e KAFKA_CFG_MESSAGE_MAX_BYTES=3072864 -e KAFKA_CFG_REPLICA_FETCH_MAX_BYTES=3072864 -v Kafka_Local:/bitnami/kafka bitnami/kafka:latest

kafka (NFS)

docker run -d --name kafka -p 9092:9092 -p 9093:9093 -e KAFKA_ENABLE_KRAFT=yes -e KAFKA_CFG_PROCESS_ROLES=broker,controller -e KAFKA_CFG_CONTROLLER_LISTENER_NAMES=CONTROLLER -e KAFKA_CFG_LISTENERS=PLAINTEXT://:9092,CONTROLLER://:9093 -e KAFKA_CFG_LISTENER_SECURITY_PROTOCOL_MAP=CONTROLLER:PLAINTEXT,PLAINTEXT:PLAINTEXT -e KAFKA_CFG_ADVERTISED_LISTENERS=PLAINTEXT://host.docker.internal:9092 -e KAFKA_CFG_BROKER_ID=1 -e KAFKA_CFG_CONTROLLER_QUORUM_VOTERS=1@localhost:9093 -e ALLOW_PLAINTEXT_LISTENER=yes -e KAFKA_KRAFT_CLUSTER_ID=kafka_local -e KAFKA_CFG_NODE_ID=1 -e KAFKA_CFG_MESSAGE_MAX_BYTES=3072864 -e KAFKA_CFG_REPLICA_FETCH_MAX_BYTES=3072864 --mount 'type=volume,source=Kafka_Local,target=/bitnami/kafka,volume-driver=local,volume-opt=type=nfs,volume-opt=device=:/home/alex/data/Kafka_Local,"volume-opt=o=addr=192.168.1.12,rw,nfsvers=4"' bitnami/kafka:latest

kafka (маппинг)

docker run -d --name kafka -p 9092:9092 -p 9093:9093 -e KAFKA_ENABLE_KRAFT=yes -e KAFKA_CFG_PROCESS_ROLES=broker,controller -e KAFKA_CFG_CONTROLLER_LISTENER_NAMES=CONTROLLER -e KAFKA_CFG_LISTENERS=PLAINTEXT://:9092,CONTROLLER://:9093 -e KAFKA_CFG_LISTENER_SECURITY_PROTOCOL_MAP=CONTROLLER:PLAINTEXT,PLAINTEXT:PLAINTEXT -e KAFKA_CFG_ADVERTISED_LISTENERS=PLAINTEXT://host.docker.internal:9092 -e KAFKA_CFG_BROKER_ID=1 -e KAFKA_CFG_CONTROLLER_QUORUM_VOTERS=1@localhost:9093 -e ALLOW_PLAINTEXT_LISTENER=yes -e KAFKA_KRAFT_CLUSTER_ID=kafka_local -e KAFKA_CFG_NODE_ID=1 -e KAFKA_CFG_MESSAGE_MAX_BYTES=30728640 -e KAFKA_CFG_REPLICA_FETCH_MAX_BYTES=30728640 --mount type=bind,source=/var/data/Kafka_Local,target=/bitnami/kafka bitnami/kafka:latest

монтируем kafka-ui

docker run --name kafka-ui -d -p 9001:8080 -e DYNAMIC_CONFIG_ENABLED=true -e KAFKA_CLUSTERS_0_BOOTSTRAP_SERVERS=host.docker.internal:9092 -e KAFKA_CLUSTERS_0_NAME=kafka_local provectuslabs/kafka-ui:v0.7.2

Важно проверить - в /etc/hosts должна быть запись - 127.0.0.1   host.docker.internal,
иначе будет ошибка в приложении по резолвингу хоста

kafka-ui запускаем: http://localhost:9001


Увеличить (при необходимости) максимальное количество символов в сообщении,
для это в настройках (Settings) топика устанавливаем параметр max.message.bytes в нужное значение.
В настройках Kafka это уже указано, также нужно в настройках клиента (Producer) установить нужные настройки, например такие:
MessageMaxBytes = 30485880,
QueueBufferingMaxKbytes = 40000
